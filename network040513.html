Setting up a Network

  • Composed of:
    • Router
    • HTTP Server (2)
    • LDAP Server
    • VPN Server
    • FTP Server
    • mySQL Server
    • Mail Server
  • Design rationale:
    • Cost Effectiveness
      • Linux based network
        • Hardware is the only cost
        • Minimal hardware required
        • Allows budget to cover backup hardware
    • Redundancy
      • every service runs on own server
        • If one service or server fails, the network's integrity is preserved
        • Failed servers can be swapped out easily
  • Server Functions
    • Router
      • Runs clearOS
      • Could be swapped out for hardware based router running clearOS
      • Easy to configure
        • No outsourcing required, just a 'tech savvy' employee.
      • Acts as a primitive firewall
      • Acts as a DNS
    • HTTP Servers
      • Serve the public and internal websites
      • One for Intranet, one for Internet
    • LDAP Server
      • Unifies login between network services
      • Holds employee info
        • e.g. Username, email etc.
    • VPN Server
      • Allows workers to access the network remotely + securely
      • Allows workers to use the network as if they were physically connected to it
    • FTP Server
      • tentative
      • Holds employee's files.
  • Intranet
    • Looking into groupware solutions
    • Groupware:
      • Software that allows multiple people to collaborate
      • Usually bundles services such as email, calendar, contacts, customer relationship management, etc.
    • Horde
      • Mature, Enterprise level groupware
      • Benefits:
        • FREE
        • Well Tested
        • Widely deployed on enterprise networks
        • Unifies network services
      • Costs:
        • Hard to configure
        • May require outsourcing for additional configuration (after we're done)
    • Custom Groupware
      • May need to code up custom groupware to fit Sam's needs
      • More flexible than Horde, can tailor suit the solution to LLTIC
